# Landing Copy - Copy Persuasivo Sin Humo

## Filosofía Central

El copy efectivo **no manipula, clarifica**. La persuasión honesta se basa en:

1. **Especificidad sobre generalidad** - "Reduce 3 horas de trabajo manual a 15 minutos" vs "Ahorra tiempo"
2. **Beneficios reales sobre promesas vacías** - Lo que el producto hace, no lo que el usuario sueña
3. **Lenguaje humano sobre jerga corporativa** - Como hablarías con un colega, no un comunicado de prensa
4. **Honestidad sobre exageración** - Si algo es bueno, no necesita ser "revolucionario"

## Antes de Escribir: Preguntas Obligatorias

Recopilar esta información antes de generar cualquier copy:

| Pregunta | Por qué importa |
|----------|-----------------|
| ¿Qué hace el producto en una frase? | Claridad fundamental |
| ¿Qué problema específico resuelve? | El dolor real del usuario |
| ¿Quién es el usuario ideal? | Tono y referencias |
| ¿Qué alternativas usa actualmente? | Diferenciación honesta |
| ¿Cuál es el resultado medible? | Especificidad del beneficio |

Si el usuario no proporciona esta información, solicitarla antes de escribir.

## Estructura de Landing Page

### 1. Hero Section

**Componentes:**
- Headline (máx 10 palabras)
- Subheadline (1-2 líneas que expandan el headline)
- CTA primario
- Elemento de prueba social opcional

**Fórmula para Headlines efectivos:**

```
[Resultado específico] + [para quién] + [sin fricción obvia]
```

**Ejemplos buenos:**
- "Facturación automática para freelancers. Sin Excel, sin dolores de cabeza."
- "Gestiona todos tus proyectos en un solo lugar. Tu equipo te lo agradecerá."

**Ejemplos a evitar:**
- "Revoluciona tu productividad con IA de última generación" (vacío)
- "El futuro del trabajo ya está aquí" (cliché sin sustancia)
- "Potencia tu negocio al siguiente nivel" (no dice nada)

### 2. Sección de Problema/Dolor

Describir el problema que el usuario reconoce, no inventar uno. Usar lenguaje que el usuario usaría al quejarse con un compañero.

**Estructura:**
```
¿Cansado de [problema específico y reconocible]?
[Descripción de 2-3 consecuencias reales]
```

### 3. Sección de Solución/Beneficios

**Regla de los 3 beneficios:** Máximo 3 beneficios principales. Cada uno debe:
- Ser verificable o demostrable
- Conectar con el problema mencionado
- Tener un "so what" claro (por qué le importa al usuario)

**Formato recomendado:**
```
[Beneficio concreto]
[Cómo funciona en 1 línea]
[Resultado para el usuario]
```

### 4. Prueba Social

**Jerarquía de credibilidad (de mayor a menor):**
1. Métricas específicas ("12,000 equipos lo usan", "4.8/5 en G2")
2. Logos de empresas conocidas (si es real y autorizado)
3. Testimonios con nombre, cargo y empresa
4. Testimonios anónimos (último recurso)

**Nunca inventar testimonios o métricas.**

### 5. Pricing

**Principios:**
- Claridad sobre el precio real (sin asteriscos tramposos)
- Comparación honesta entre planes
- Si hay limitaciones, mencionarlas

### 6. CTA Final

El CTA debe reflejar el siguiente paso real, no prometer resultados.

**Buenos CTAs:**
- "Prueba gratis 14 días" (si es verdad)
- "Empieza tu proyecto"
- "Crea tu cuenta"

**Malos CTAs:**
- "Transforma tu negocio" (eso lo hace el producto, no el clic)
- "Únete a la revolución" (cringe)

## Frameworks de Copy

Consultar `references/frameworks.md` para guías detalladas de:
- **PAS** (Problem-Agitate-Solution) - Para productos que resuelven dolores claros
- **AIDA** (Attention-Interest-Desire-Action) - Para productos nuevos o desconocidos
- **BAB** (Before-After-Bridge) - Para mostrar transformación

La skill elige el framework más adecuado según el contexto. Si el usuario tiene preferencia, respetarla.

## Anti-Patrones: Lo Que NUNCA Escribir

Consultar `references/anti-patrones.md` para la lista completa de frases prohibidas y por qué evitarlas.

**Resumen rápido de palabras/frases a evitar:**
- "Revolucionario", "disruptivo", "game-changer"
- "Potencia", "maximiza", "optimiza" (sin contexto específico)
- "Solución integral", "ecosistema", "sinergia"
- "De última generación", "cutting-edge", "state-of-the-art"
- "Lleva tu X al siguiente nivel"
- Cualquier promesa que no se pueda demostrar

## Formato de Entrega

Al generar copy, entregar en este formato:

```markdown
## [Sección]

**Opción A:**
[Copy]

**Opción B:**
[Copy alternativo]

**Notas:** [Contexto sobre las decisiones tomadas]
```

Siempre ofrecer al menos 2 opciones para elementos clave (headlines, CTAs) para que el usuario elija según su tono preferido.

## Checklist Final

Antes de entregar cualquier copy, verificar:

- [ ] ¿Cada afirmación es verdadera o verificable?
- [ ] ¿Un usuario real diría estas palabras?
- [ ] ¿El beneficio es específico, no genérico?
- [ ] ¿El CTA refleja la acción real?
- [ ] ¿Evité toda la jerga de la lista de anti-patrones?
- [ ] ¿Un competidor podría usar exactamente este copy? (si sí, es muy genérico)
